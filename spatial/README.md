# 空間図形シミュレータ「どこから撮ったのかな？」

中学1年 数学（東京書籍「新編 新しい数学1」第6章 空間図形 p.197／第4時）の
**平面が1つに決まる条件**を体験するための自作ツールです。単一の `index.html` だけで動きます。

## 中身（2つのタブ）
1. **① 撮影位置をさがそう** … ピサの斜塔（＝斜めに立てた1本の直線）を撮った「お手本の写真」に
   ぴったり重なるカメラの位置・高さ・向き・仰角を探す。**「真上から（地図）」**と**「真横から（側面図）」**の
   2つの補助図で、位置・向き・高さ・仰角が1対1で対応することを体感し、「視点1点＋対象（直線）で
   見える景色（平面）がただ1つに決まる」ことに気づかせる。〔導入〕
   - 実際の**ピサ「奇跡の広場」**を3D・地図に再現。
   - **「▶ ぐるっと1周（動画）」ボタン**で、塔の周りをカメラが自動で回り込む（＝視点が変われば写真も変わる）。
   - **お手本の写真をえらぶ**：教科書 p.197 の3枚（ア・イ・ウ）から選ぶと、その写真の撮影位置が目標（★）になる。写真は `画像/kadai_a.png`（ア）・`kadai_i.png`（イ）・`kadai_u.png`（ウ）。
   - 地図に**位置の候補 ア・イ・ウ**（南・西・北）を表示。立つ位置で塔の傾きの見え方（右・正面・左）が変わることを確かめられる。
2. **② 平面が決まる条件** … 点が1個／2個／3個（一直線上）／3個（一直線上にない）で
   板（平面）がぐらぐら動くか固定されるかを、スライダーで確かめる。〔展開の補助〕

## 使い方
- ファイルをブラウザで開くだけ（Three.js は CDN から読み込みます。ネット接続が必要）。
- タブレット配布時は、GitHub Pages 等に置いて **QRコード** で配るのが簡単です。

### GitHub Pages で公開する手順（例）
1. GitHub で新しいリポジトリ（例：`spatial-figure`）を作成。
2. この `index.html` をアップロード（またはpush）。
3. リポジトリの Settings → Pages → Branch を `main` / `root` にして保存。
4. 数分後に `https://<ユーザー名>.github.io/spatial-figure/` で公開。
5. そのURLのQRコードを作り、ワークシートやスライドに貼る。

## 授業での位置づけ
- 導入(15分)：タブ①で撮影ポイントを探索 → 「1つに決まる」感覚をつかむ。
- 展開(20分)：タブ②＋具体物（指先・下敷き）で「一直線上にない3点」を確かめる。
- まとめ(15分)：体験と結びつけて条件を言語化。

## フォルダ構成
- `index.html` … 本体（これだけで動く）
- `画像/` … お手本写真 `kadai_a.png` `kadai_i.png` `kadai_u.png`

## 調整メモ
- 3枚の撮影位置は `index.html` 内の `SCEN`（a/i/u それぞれ x, z, h, az, pt）で変更できます。
- 斜塔の傾きは `buildTower()` の `g.rotation.z` で調整できます。
